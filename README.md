# CNN HPC AI Agent — Explorer @ Northeastern

Fully autonomous self-healing ML pipeline. One command — it does everything.

## What it does

1. Detects the Explorer environment, loads modules, creates conda env automatically
2. Generates a synthetic 3-class image dataset (1500 samples, 32×32 RGB)
3. Writes and fixes `submit_job.sh` with correct Explorer module names
4. Submits a SLURM job to Explorer (GPU partition)
5. Trains a SmallCNN with **Optuna HPO** (20 trials × 30 epochs each)
6. Monitors the job every 30s — auto-heals errors and resubmits if needed
7. Saves all results — per-epoch logs for every trial + ranked summary
8. Calls **Groq AI (Llama 3.3 70B — free)** and auto-applies optimizations to `train_cnn.py`
9. Re-runs with the optimized config automatically

## Usage

### Run as a SLURM job (recommended — never gets killed)

```bash
sbatch run_agent.sh
tail -f agent_run.log
```

### Or run in background on login node

```bash
nohup python agent.py > agent_run.log 2>&1 &
tail -f agent_run.log
```

## Auto-heal capabilities

The agent automatically fixes these errors without human intervention:

| Error | Auto-fix applied |
|-------|-----------------|
| Wrong conda env path | Detects `~/.conda/envs/` directly |
| Wrong module names | Scans `module avail` and picks correct ones |
| Optuna import errors | Patches `train_cnn.py` automatically |
| CUDA driver mismatch | Falls back to CPU gracefully |
| Missing directories | Creates them automatically |
| OOM errors | Reduces batch size and resubmits |
| Job timeout | Reduces `n_trials` and `max_epochs` and resubmits |

Max retries: 5. Each failure is diagnosed, patched, and resubmitted automatically.

| Dropout | 0.1 → 0.5 |
